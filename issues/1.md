### Background
This project would benefit from a terminal user interface (TUI) that's as modern and rich as btop or mactop, which use animations, widgets, real-time resource monitors, and detailed stats to enhance user experience.

### Key Research Insights from btop/mactop:
1. Both utilities use **dynamic terminal rendering** via ANSI escape codes to create animations and refreshed widgets without relying on external libraries.
2. UI elements like graphs, memory usage bars, and process lists are rendered interactively with minimal CPU overhead.
3. They rely on a virtual screen concept to redraw only "dirty" (i.e., changed) regions, optimizing terminal redraw frequency.
4. **Real-time system stats** are collected with platform APIs (e.g., `/proc`, `sysctl`), enabling fluid updates.
5. **Interactive navigation** is key, allowing users to browse panels, sort data, and interact dynamically without mouse support.
6. Color highlights (e.g., CPU graphs, theme choices) use ANSI 256-color or TrueColor sequences for full terminal integration.

### Proposal
I propose the following changes:
1. Refactor the TUI rendering mechanism to simulate btop's approach:
   - Build a "virtual screen buffer" to optimize redraws.
   - Use ASCII and Unicode glyphs combined with block charts for smoother graphs and stats.
2. Add dynamic widgets:
   - Include CPU, memory, disk stats, a process tree, and task monitor views.
   - Let users toggle or customize visible widgets.
3. Enhance interactivity:
   - Functional keyboard shortcuts for navigation.
   - (Optional) Mouse-click integration for widget expansion/detail views.
4. Include theming support.
   - Begin with a dark theme and monochrome option.
